# ☁️ AWS Services – Complete Guide (Interview Ready)

---

## 📌 Introduction

Amazon Web Services (AWS) provides a wide range of cloud services that help in building scalable, reliable, and secure applications.

These services are broadly categorized based on their functionality.

---

## 🧩 1. Compute Services

### 📌 Purpose:

Used to run applications and workloads.

### 🚀 Services:

* EC2 (Elastic Compute Cloud)
  → Virtual servers in the cloud

* Lambda
  → Serverless computing (run code without managing servers)

* Elastic Beanstalk
  → Platform for deploying apps without managing infrastructure

---

### 💡 Interview Point:

Compute services provide processing power for applications.

---

## 💾 2. Storage Services

### 📌 Purpose:

Store and retrieve data.

### 🚀 Services:

* S3 (Simple Storage Service)
  → Object storage (images, videos, backups)

* EBS (Elastic Block Store)
  → Storage for EC2 instances

* Glacier
  → Long-term archival storage

---

### 💡 Interview Point:

S3 is highly durable and scalable object storage.

---

## 🗄 3. Database Services

### 📌 Purpose:

Store structured and unstructured data.

### 🚀 Services:

* RDS (Relational Database Service)
  → Managed SQL databases (MySQL, PostgreSQL, etc.)

* DynamoDB
  → NoSQL database

* Aurora
  → High-performance relational database

---

### 💡 Interview Point:

RDS is managed, DynamoDB is serverless NoSQL.

---

## 🌐 4. Networking Services

### 📌 Purpose:

Manage communication between resources.

### 🚀 Services:

* VPC (Virtual Private Cloud)
  → Private network in AWS

* Route 53
  → DNS service

* CloudFront
  → Content Delivery Network (CDN)

---

### 💡 Interview Point:

VPC provides isolation and security.

---

## 🔐 5. Security & Identity

### 📌 Purpose:

Control access and secure resources.

### 🚀 Services:

* IAM
  → Access control

* Cognito
  → User authentication for apps

* Shield
  → DDoS protection

---

### 💡 Interview Point:

IAM follows the principle of least privilege.

---

## ⚙️ 6. DevOps & Deployment

### 📌 Purpose:

Automate development and deployment.

### 🚀 Services:

* CodeCommit
  → Git repository

* CodeBuild
  → Build service

* CodeDeploy
  → Deployment automation

* CodePipeline
  → CI/CD pipeline

---

### 💡 Interview Point:

AWS provides full CI/CD ecosystem.

---

## 📊 7. Monitoring & Logging

### 📌 Purpose:

Track performance and logs.

### 🚀 Services:

* CloudWatch
  → Monitoring and alerts

* CloudTrail
  → Track API activity

---

### 💡 Interview Point:

CloudWatch is used for metrics and alarms.

---

## 🤖 8. AI & Machine Learning

### 📌 Purpose:

Build intelligent applications.

### 🚀 Services:

* SageMaker
  → Build & train ML models

* Rekognition
  → Image and video analysis

* Lex
  → Chatbots

---

---

## 📦 9. Application Integration

### 📌 Purpose:

Connect different services and systems.

### 🚀 Services:

* SQS (Simple Queue Service)
  → Message queue

* SNS (Simple Notification Service)
  → Pub/Sub messaging

---

---

### Scenario:

Deploy a web application on AWS

### Solution:

* EC2 → Run backend

* CloudFront → CDN
